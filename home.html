Senior Design Project

Intrusion Detection System for Industrial Control Systems with a focus in passenger vehicles

Description: My senior design project is to design an Intrusion Detection System for the CAN Bus protocol that is used to send signals throughout passenger vehicles. The process for designing our IDS follows a few steps. First we must set up a CAN channel as a testbed where we will deploy our IDS, this is followed by introducing attack vectors to the environment for our IDS to detect, and lastly we test different aspects like efficiency and accuracy.

Role: At the beginning, I am responsible for setting up the initial testbed using components that can emulate the CAN signals being sent in real vehicles. When it comes to the IDS, I have looked at typical CAN frames, both safe and malicious, to set up the rules the IDS will use when analyzing the frames that come in.

Skills/Knowledge: I learned a lot about how CAN works and it gave me an insight into Industrial Control Systems which I had little experience with prior to the project. I also had the chance to improve my Python programming skills when designing the IDS and got to use new libraries that can deal with CAN frames. Aside from technical skills, being able to work on an engineering problem with a group was a positive experience for learning more intrapersonal skills when dealing with group projects.

Big Picture Contribution: Our project aims to bring greater security to passenger vehicles that everyone drives today by alerting users and producers of any potential intrusions that may be happening within the systems of the vehicle.

CPRE 381 --- Processor Design

Description: The focus of this project was to design a processor that could execute plenty of assembly commands. The processor needed to be capable of doing math necessary for these commands as well as making actions in memory to store, move, or delete memory within the caches and storage. Each component within the processor was designed by my partner and I from the simple components of AND gates all the way up to complex adders.

Role: I was responsible for creating the high level design of the processor and this meant having a deep understanding of how the components work together and what signals need to go where. Along with high level design, I was also required to design half of the components necessary for our processor to properly function and execute all of the commands.

Skills/Knowledge: This project gave me a deeper understanding of the inner workings of computers and how assembly language is used to execute programs. I also learned VHDL for this project, which is a skill I would not have gotten to work with was it not for this class. This project also required a lot of debugging when testing different commands, so being able to learn better strategies and techniques for debugging is something I will continue to use while programming.

COM S 309 --- Auto Battler Game

Description: This group project was one of my first experiences with application development and programming with a team. Our team decided to create and auto battler game where two players would play against each other by creating a squad of characters which each have their own abilities and stats. Then using those stats they would face off and the winner of each round would get a point. Every round, each player can build upon their team in order to make it stronger in order to beat their opponent. Each player could create an account that would be saved in a database and their stats would accompany their account and could be viewed in game.

Role: My role for this project was front-end design and game functionality. This involved making sure our game would navigate correctly, look appealing, and that the game was playable. In order to make the game function correctly, I had to ensure teams could be properly sent to the server and it would receive the result of each game, as well as making sure each encounter between characters and their stats or abilities would work properly and execute in the correct order.

Skills/Knowledge: The biggest piece of knowledge that I gained was software development planning. Using gitlab and creating/resolving issues I better understood the planning and organization that is necessary for a project of this scale to stay on schedule and fulfill the requirements. This also greatly improved my skills programming with Java as I did many things I had not done before in that language, like making connections to the back-end of our project as well designing functional, attractive user interfaces.
